Greedy choice

Different paradigm, worse answer

A bad local-walk heuristic on the graph A-B:1, B-C:9, A-C:2, C-D:1, B-D:2 can pick A-B, B-C, C-D for total 11. Kruskal picks A-B, C-D, A-C for total 4 because it always takes the lightest safe edge globally, not just the next edge from one vertex.

Overview Section

Kruskal's algorithm builds a minimum spanning tree by sorting all edges from lightest to heaviest, then scanning that list once. Every time it finds an edge whose endpoints are in different components, it accepts that edge because it is safe by the cut property.

This tool shows three views at the same time: the graph itself, the sorted edge order, and the Union-Find forest. The teaching goal is to make every step visible instead of treating cycle detection as a black box.

When to use it: weighted, undirected, connected graphs, especially when edges are easy to sort or when the graph is sparse.

Worked Example
  1. Load the preset Balanced Five. The tool sorts the edges to get (B-C,1), (D-E,1), (B-D,2), (A-C,3), (A-B,4), (C-D,4), (C-E,5).
  2. It accepts B-C and D-E because both connect separate components.
  3. It accepts B-D because it joins the two components {B,C} and {D,E}.
  4. It accepts A-C because A is still separate. Now all five vertices are connected with 4 edges, so the MST is complete.
  5. The final MST weight is 1 + 1 + 2 + 3 = 7.
Key Concepts and Common Mistakes
Cut PropertyThe lightest edge crossing any cut is safe to add to some MST. This is the visual proof behind Kruskal's greedy choice.
Cycle RejectionIf both endpoints already have the same root, that edge closes a cycle and must be skipped.
Common MistakePicking the lightest edge adjacent to the last chosen edge is not enough. Kruskal compares across the whole sorted list.
Worst CaseDense graphs maximize E, so sorting dominates. That is why the dense preset is the worst-case demonstration here.
